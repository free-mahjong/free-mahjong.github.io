free_mahjong设计文档

理念

使我萌生创建这样一个自由化的麻将软件的初衷是以下两点。一是目前市面上的麻将软件规则基本上全是固定的，而一些小众规则（如本人所在的小破十八线城市）往往不被支持；二是线上的麻将往往会安排牌堆（也许是出于想让用户打的爽的目的），这反而在一定程度上降低了我的体验。因此，我决定自己写一款麻将软件，采用完全开放式的形式，彻底颠覆传统麻将软件，顺便解决牌堆安排问题。

何为完全开放式？我的理解是，它完全没有任何规则。那这样的麻将要如何打？答案是——这依赖于这一桌的四名玩家。也就是说，麻将的规则将完全因玩家而异，软件本身并不作出任何基于规则的限制。

举个例子，假如说我的对面打了一个红中，而我的手里有两个红中，那一般的麻将软件肯定就开始读条了，询问我要不要碰，而我的上家却不会有读条——因为我的优先级更高。那这样就会使得我的上家推测出场上有人有一对红中，这固然可以解释成feature，可是我并不觉得所有人都认为这是一个很好的feature——至少我自己便不这么认为。那么，我希望的样子是什么呢？

我的对面打了一个红中，可是，我却不会有读条！事实上，所有人都没有读条！我的上家可以自由地摸子并打出，我也可以同时把我的两个红中亮出来然后说我要碰，这听起来很古怪，可是这就是四个人凑一桌打线下麻将时的真实情况——而我要做的不过是将这一点在线上实现罢了。或许说到这里，大家就明白了，我所谓的自由麻将，无非是将线上的麻将线下化，而我最终的设计目标也正是使得我的麻将软件能几乎完全模拟线下麻将的体验。为了做到这一点，需要支持许多听起来匪夷所思的功能，但是我觉得，如果真能实现完全“面麻”的体验，这些都是值得的。

功能列表

  • ui与操作设计
碎碎念版

  • 每个人的面前都有一块区域（称为面前区）可供玩家自由放置麻将子，也即，麻将并不一定要强行排布成一排，而是可以“到处都是”（不过不能摆到其他玩家的面前区或者公共区），并且，所有玩家的面前区将以相应的视角完全呈现在其他玩家屏幕上。这样的设计可以最大程度上模拟面麻体验。

  • 摸子时，需要玩家从牌墙中拖动对应的子到自己的面前区，而没有自动摸取键。

    • 杠直接拖最后那个子就好了

  • 出子时，需要玩家将对应的子从自己的面前区拖动到公共区。

  • 吃子时（包括吃、碰、明杠），需要将自己的牌点开，再将所吃的子从公共区拖动到自己的面前区。

    • 对于“吃”而言，有些人或许也会习惯性丢掉用来吃的两个子，直接将这两个子打到公共区也可。anyway，这将是很自由的。

  • 暗杠时，需要将自己的牌盖上

    • 对于点开与盖上，目前的设计计划是，当一个子是立起来的时候，左键点开，右键盖上。当一个子是点开的时候，右键立起来。当一个子是盖上的时候，左键立起来。

  • 撤回

    • 这一点是很有意义的，因为撤回操作在面麻中经常碰到。比如之前所举的俩红中例子，如果我的上家急了，直接摸子了，那我再碰，我的上家将面临撤回的情况。

    • 目前的设计理念是，不提供任何一键撤回的按键，而是利用游戏本身的高自由度，让玩家做相反操作。例如我的上家在我碰了之后，他把他摸的子从他的面前区拖回到公共区，我再打，然后下家再重新从公共区摸回我上家误摸的子。

      • 在这个撤回例子中，撤回的过程类似于普通的打子。这样或许无法与正常出子区分开，因此我计划继续采用左右键的方式区分。正常的打子使用左键拖动到公共区，这样的子在进入公共区后自动翻开，撤回的子使用右键拖动到公共区，这样的子在进入公共区后自动盖上。
      • 对于其他的撤回情况，大约也是类似的，根据之前的设计所支持的特性，任何过程都是可逆的。

  • 关于胡牌、炸胡与相公

    • 我并不打算从任何代码的角度解决这些问题。因为我的理念就是完全依赖玩家——事实上，我的代码连玩家在干什么都不知道——他们或许在打宜春麻将规则，或许在打日麻，或许就是乱玩，但无论如何，这些都是允许的，一切都依赖于玩家。这也就是说，一旦有人声称自己胡了，ta必须把自己的所有牌都点开，让大家都瞧瞧，才能确定确实胡了，而我的代码不做任何判断。对于炸胡与相公更是如此，完全依赖于玩家自觉。比如说，如果有人全点开之后发现炸胡了，那ta就自己把牌全点回去，然后假装无事发生继续打；如果打着打着有人相公了，软件也不会提供任何程度上的检测与提示，完全依赖人工发现。而相公之后该玩家必须摸啥打啥，这也依赖于玩家自觉。

条理版（API版）

  • 桌面分为1个公共区、4个面前区、4个牌墙

    • 每个人可以操作的区域为自己的面前区、公共区和牌墙

  • 牌墙

    • 左键点击牌墙并拖动至自己的面前区：从牌墙中获取当前点击位置对应的子，并放置在面前区的相应位置上。
    • 左键点击牌墙并拖动至其他区域：无效摸牌，子回原位。
    • 右键点击牌墙并拖动至自己的面前区：从牌墙中获取当前点击位置对应的子以及紧随其后的3个子，并放置在面前区的相应位置上，模拟初始一次摸4个子的情况。
    • 右键点击牌墙并拖动至其他区域：无效摸牌，子回原位。

  • 公共区

    • 左键点击公共区并拖动至自己的面前区：将公共区中对应位置的子拖动到自己的面前区，并放置在面前区的对应位置上（默认翻开）
    • 其他：无效操作

  • 面前区

    • 左键点击自己面前区内的子：

      • 该子处于立起来的状态：使其翻开
      • 该子处于盖上的状态：使其立起来

    • 右键点击自己面前区内的子：

      • 该子处于翻开的状态：使其立起来
      • 该子处于立起来的状态：使其盖上

    • 左键点击自己面前区内的子并拖动到面前区其他位置：移动该子（保持翻/盖状态不变）

    • 左键点击自己面前区内的子并拖动到公共区：打出该子（默认翻开）

    • 右键点击自己面前区内的子并拖动到公共区：撤回该子（默认盖上）

      • 撤回的子将自动按照序号归位到牌墙对应位置

  • 通信

    • client

      • 根据用户输入的服务器IP与端口号创建一条socket连接，用于传输用户操作。

    • server

      • 同步所有用户的操作

      • 洗牌、生成牌堆

        • 洗牌操作需要4名玩家投票决定，具体ui之后再看

      • 掷骰子

        • 任意一名玩家想掷骰子就能掷骰子

  • 语音聊天

    • 这一点或许是必要的。因为作为一个完全自由化的麻将软件，其不能像一般的online游戏那样全程闭麦而基本不影响正常游戏，否则将可能使得游戏过程变得极其奇怪，互相冲突而不理解对方意图。
    • 可能有难度，实在不行就搭配已有的实时聊天软件，看延迟是否在可接受的范围之内

  • 其他特色功能

    • 统计胜负场、结算筹码、购买菜篮子

    • 砸蛋/送花/扔拖鞋/etc.

    • 发送快捷语音

      • 三国杀语音包

      • 老同学经典语音包

        • 比**还爽
        • 我已经听牌了，小心点！

      • 其他自定义语音

    • etc.

技术栈

限于本人的技术水平，我目前只计划开发windows客户端。我计划采用Qt来写前端，python后端。后端将架设在本人的服务器上，以便没有公网IP的玩家也能够进入游戏。当然，我也计划在Qt前端中加入socket通信模块，以便那些有公网IP的玩家可以直接将自己的主机作为服务器，以降低延迟。
